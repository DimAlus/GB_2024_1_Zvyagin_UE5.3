# Геймплей

Игрок является бестелесной тучкой _(это должен быть начальный облик, не реализовано)_,
которая может вселяться во врагов (`контроль`).

### Цель 
Пройти какое-то количество уровней.

### Способности

* Способности игрока
	- Вселение в противника (`Q`)
* Способности противников
	- Огненный шар (`ЛКМ`)
	- Лазер из глаз (`ПКМ`)

При вселении в противника игрок получает способности противника.

После снятия видео после "выселения" из противника, тот становится союзником

### Условие проигрыша

Игрок проигрывает в случае полной потери здоровья текущего тела.

## Компоненты

Поведение, данные, возможности персонажа определяются набором
его компонентов. Компоненты дают интерфейс области, с которой работают,
другим компонентам запрещено определять поведение этой области.

Для каждого типа компонента существует инициализатор, 
структура, определяющая характеристики поведения. 
Компонент же определяет тип поведения

**Например:** Условно, 
компонент _MovementOnTheGround_ будет перемещать персонажа по земле,
а _MovementFly_ по воздуху. 
_MovementComponentInitializer_ будет в свою очередь 
будет определять скорость перемещения.

Каждый компонент имеет базовый класс, который дает интерфейс 
и не делает ничего. (_Socium исключение, надо переписать_)

### InputComponent
Реакция на триггеры пользовательского ввода

### CameraComponent
Управление камерой игрока

### MovementComponent
Перемещение пешки в мире, работа с ее физическим телом

### HealthComponent
Здоровье и смерть

### WeaponComponent
Управление вооружением / навыками (релокация тоже является оружием)

Могут спавнить снаряды.

### SociumComponent
Самоопределение пешки в мире, ее мировоззрение и зрение.

Дает возможность пешке стать ячейкой общества.
